## Why Does L1 Penalty on Sigmoid Gates Encourage Sparsity?

The gate for each weight is defined as:

```
gate = sigmoid(score)  ∈ (0, 1)
```

The sparsity loss is the **L1 norm** of all gate values:

```
SparsityLoss = Σ |gate_i| = Σ gate_i       (since gates > 0 always)
```

The total loss becomes:

```
Total Loss = CrossEntropyLoss + λ × SparsityLoss
```

**Why L1 and not L2?**  
L2 (sum of squares) penalizes large values heavily but produces many small-but-nonzero values — it "shrinks" but doesn't "zero out."  
L1 (sum of absolute values) has a **constant gradient** (`±λ`) regardless of the value's magnitude. This constant pull toward zero is strong enough to push small gate values **all the way to 0**, creating true sparsity.  
This is the same reason Lasso regression produces sparse coefficients while Ridge does not.

In practice, the optimizer sees: "every active gate costs me `λ` in loss per unit value — so if a gate isn't helping accuracy, kill it."

## Gate Value Distribution Plot

The plot `gate_distribution.png` (generated by the script) shows gate values for the best model (λ = `1e-5`).

**What a successful result looks like:**
- **Large spike near 0** — majority of weights pruned (gates ≈ 0)
- **Smaller cluster away from 0** — surviving important connections (gates ≈ 0.5–1.0)
- **Bimodal distribution** — clear separation between "dead" and "alive" gates

This bimodal shape confirms the network learned to self-organize: it kept only the connections that contributed to classification and zeroed out the rest.

## Key Design Choices

**1. Sigmoid as gate activation**  
Bounds gates to (0,1). Differentiable everywhere — gradients flow to `gate_scores` cleanly.

**2. Gradients flow through both parameters**  
`pruned_weight = weight * sigmoid(gate_scores)`  
Autograd differentiates through the multiplication, so:  
- `∂Loss/∂weight` updates the weight values  
- `∂Loss/∂gate_scores` updates whether the weight is active at all

**3. Gate initialization at 0**  
`sigmoid(0) = 0.5` → gates start at 50% — neutral, letting training decide what to prune.
